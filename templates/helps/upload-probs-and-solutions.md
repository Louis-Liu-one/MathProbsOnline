# 上传题目与题解相关事宜

## 摘要

本篇帮助介绍了有关上传题目、题解的若干注意事项和可能遇到的问题。

## 相关事宜

你可以在[这里]({{ url_for('upload_prob') }})上传自己的题目到我们的云端，或在题目主页点击侧边栏的“上传题解”以上传你的题解。除此之外，你也可以编辑你的题目或题解。

### 题目编号的填写

上传题目前，必须填写*有意义、有区分度*的题目编号，不超过 16 个字符，建议全部使用 ASCII 字符。上传后，题目路径在 `/probs/<题目编号>`。

题目编号建议遵循统一的格式，例如物理题是四位十六进制整数加上前缀 `PHY`，如 `PHY1234`。除非经过管理员的同意，尽量不要使用官方题目的编号格式，即 4 位 16 进制整数。不要使用过短的编号，建议编号长于 4 位。

成功上传题目后不能修改编号。如果题目有大的改动，请删除题目后重新上传。

### 题目标签的填写

题目标签是一个 CSV 格式的字符串，标签各项用英文半角逗号（“,”）分隔开。除非有必要，否则不要自创新标签，而是尽量使用已有的标签。在[这里]({{ url_for('labellist') }})查看所有已有的标签。

### 答案的填写

题目答案是多个测试点，每个测试点包含*变量*和*正确结果*两部分。
* *变量*，指用户输入表达式中一些未知数应该代入的值。例如，用户提交表达式 $x + \sqrt y$，测试点变量为 `{"y": 4}`，则代入后结果为 $x + 2$。变量需要按照 JSON 字典的格式输入，没有变量时置空或填`{}`。
* *正确结果*，是指将用户输入表达式代入给定变量值后应该得出的值，例如上例中用户计算得的结果为 $x + 2$。若题目提供的正确结果与计算得的结果相同，则此测试点判为正确。正确结果中可以有未知数。

### 图片上传

你可以为题目、题解上传图片。上传后，图片的路径在 `/images/<（题解所属的）题目编号>/<图片文件名称>`。上传后要在正文中用 Markdown 语法插入图片。例如，若题目编号为 `ABCD`，上传的图片名称为 `image.svg`，则要使用
```
![ImageAlt](/images/ABCD/image.svg)
```
在文中插入图片。

#### 推荐的图片格式

对于几何矢量图，我们推荐 *SVG 格式*；对于一般图像，我们推荐 *JPEG 格式*。**不推荐**上传 PNG 格式的或过大的图片，这会使加载题目时速度较慢。

## 常见问题与解答

### 为什么我的图片没有显示？

上传图片后还要在正文中用 Markdown 语法插入图片，否则即使图片已经上传到数据库中，也不会显示。请勿重复上传图片。

### 如何由 LaTeX 生成 SVG？

可以先将 $\LaTeX$ 编译为 DVI 文件，再用 `dvisvgm` 工具将 DVI 文件转换为 SVG 文件。$\LaTeX$ 文件建议采用 `standalone` 文档类。这里附上 `makefile`：
```
%.dvi: %.tex
    latexmk -auxdir=.aux -dvi $<
%.svg: %.dvi
    dvisvgm -n $<
```
直接使用 `make <你的 LaTeX 文件名>.tex` 即可生成 SVG 文件。

需要注意的是，`dvisvgm` 工具可能会出现问题
```
processing of PostScript specials is disabled (Ghostscript not found)
```
这时，应该使用其 `--libgs=` 参数配置 GhostScript 动态库。如果你的电脑上没有 GhostScript 动态库，可以使用 Brew 安装：
```
brew install ghostscript
```
此时，你的 GhostScript 动态库*可能*在：
```
/usr/local/Cellar/ghostscript/10.05.1/lib/libgs.10.05.dylib
```
具体位置取决于你的 Brew 安装目录和 GhostScript 版本。
